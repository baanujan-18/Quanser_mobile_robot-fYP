## **Purpose of This Phase**

SLAM Toolbox requires laser scan data for 2D mapping.

Since the robot uses a Kinect v1 depth camera instead of a 2D LiDAR sensor, the Kinect depth image must be converted into a fake laser scan.

The conversion pipeline is:

```text
Kinect depth image
        ↓
/kinect/depth/image_raw
        ↓
Camera info fixer
        ↓
/kinect/depth/camera_info_fixed
        ↓
depthimage_to_laserscan
        ↓
/scan
```

---

## **Step 1 - Install depthimage_to_laserscan**

```bash
sudo apt install -y ros-humble-depthimage-to-laserscan
```

---

## **Step 2 - Create Camera Info Fixer Script**

The Kinect v1 depth node published camera info, but it was not complete enough for `depthimage_to_laserscan`.

Therefore, a camera info fixer script was created.

Script location during testing:

```text
~/fix_kinect_camera_info.py
```

The script subscribes to:

```text
/kinect/depth/camera_info
```

and republishes corrected camera info to:

```text
/kinect/depth/camera_info_fixed
```

---

## **Step 3 - Run Kinect Depth Node**

### **Tab 1 - Kinect Depth Node**

```bash
cd ~/Ros2-KinectV1
source /opt/ros/humble/setup.bash
source install/setup.bash
sudo modprobe -r gspca_kinect
ros2 run ros2_kinect_depth depth_node
```

Expected Kinect topics:

```text
/kinect/depth/image_raw
/kinect/depth/camera_info
```

## **Step 4 - Run Camera Info Fixer**

### **Tab 2 - Camera Info Fixer**

During actual testing, the script was run from the home directory:

```bash
source /opt/ros/humble/setup.bash
python3 ~/fix_kinect_camera_info.py
```

In this GitHub repository, the same script is stored at:

```text
ros2_ws/src/qbot2_perception/scripts/fix_kinect_camera_info.py
```

So it can also be run from the repository location using:

```bash
source /opt/ros/humble/setup.bash
python3 ros2_ws/src/qbot2_perception/scripts/fix_kinect_camera_info.py
```

Expected fixed camera info topic:

```text
/kinect/depth/camera_info_fixed
```

## **Step 5 - Create depth_to_scan Launch File**

Launch file created at:

```text

```

The launch file uses the following remappings:


## **Problems Faced**

* The Kinect v1 camera info was incomplete.
* `depthimage_to_laserscan` required valid camera info.
* `/scan` did not publish correctly until the camera info fixer was used.

---

## **Solution**

A Python camera info fixer script was created.

It corrected the camera intrinsic parameters and republished the camera info to:

```text
/kinect/depth/camera_info_fixed
```

Then `depthimage_to_laserscan` used this fixed topic and successfully published:

```text
/scan
```
