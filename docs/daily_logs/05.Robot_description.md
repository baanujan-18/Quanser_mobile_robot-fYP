## **Purpose of This Phase**

ROS 2 needs to understand the physical structure and coordinate frames of the robot.

The URDF file defines the relationship between:

* Robot base
* Robot body
* Kinect camera
* Kinect depth frame

This is important because SLAM Toolbox and Nav2 need correct TF frames to understand where the sensor is mounted on the robot.

## **Important Note**

The frame name:

```text

```

was used because the depth-to-laserscan launch file also uses:

```text
output_frame: kinect_depth_frame
```

This keeps the sensor frame consistent between the URDF and the `/scan` topic.

## **Problems Faced**

* The sensor frame name had to match the depth-to-laserscan output frame.
* If the URDF frame name and `/scan` frame do not match, SLAM may fail because TF will be incorrect.

---

## **Solution**

The depth frame was named:

```text
kinect_depth_frame
```

in both:

```text
qbot2.urdf
```

and:

```text
depth_to_scan.launch.py
```
