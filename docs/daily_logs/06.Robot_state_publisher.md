## **Purpose of This Phase**

Robot State Publisher is used to publish the coordinate frames of the robot.

It reads the URDF file from the `qbot2_description` package and publishes the relationship between robot links such as:

```text
base_footprint
base_link
camera_link
kinect_depth_frame
```

This is important because SLAM Toolbox needs correct TF frames to understand where the Kinect depth sensor is mounted on the robot.


## **Problems Faced**

* The URDF frame name had to match the depth-to-laserscan output frame.
* If `kinect_depth_frame` was missing or incorrect, `/scan` data could not be correctly linked to the robot base.

---

## **Solution**

The URDF file and depth-to-laserscan launch file were kept consistent using the same frame name:

```text
kinect_depth_frame
```

This allowed the SLAM system to correctly understand the Kinect sensor position.
