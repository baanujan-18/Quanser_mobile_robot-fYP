# Day 03 - Kinect v1 Setup

## **Phase**

**Phase 5 - Install Kinect Driver**

---

## **Work Completed**

* Installed Kinect v1 required packages using `libfreenect`.
* Checked Kinect hardware detection using `lsusb`.
* Confirmed that Kinect v1 was detected as Xbox NUI devices.
* Cloned the `KinectV1-Ros2` package.
* Built the Kinect ROS 2 package while skipping the microphone package.
* Applied the Kinect USB kernel module fix using `sudo modprobe -r gspca_kinect`.
* Launched the Kinect depth node successfully.
* Verified that Kinect depth data was published through ROS 2.

---

## **Reason for Change from Original Plan**

OpenNI2 was not used because it did not work correctly with Kinect v1 in this setup.

The working solution was to use:

```text
KinectV1-Ros2
```

This package uses `libfreenect` directly and worked correctly with the Xbox Kinect v1 sensor.

---

## **Commands Used**

### **Install libfreenect**

```bash
sudo apt install -y libfreenect0.5 libfreenect-bin libfreenect-dev freenect
```

---

### **Check Kinect Hardware**

```bash
lsusb
```

Expected Kinect-related devices:

```text
Microsoft Corp. Xbox NUI Camera
Microsoft Corp. Xbox NUI Audio
Microsoft Corp. Kinect for Windows NUI Motor
```

---

### **Clone KinectV1-Ros2 Package**

```bash
mkdir ~/Ros2-KinectV1
cd ~/Ros2-KinectV1
git clone -b ros2-humble https://github.com/SriharshaShesham/KinectV1-Ros2.git .
```

---

### **Build KinectV1-Ros2**

```bash
cd ~/Ros2-KinectV1
source /opt/ros/humble/setup.bash
colcon build --symlink-install --packages-skip ros2_kinect_mic_node
```

---

### **Source Kinect Workspace**

```bash
source ~/Ros2-KinectV1/install/setup.bash
echo 'source ~/Ros2-KinectV1/install/setup.bash' >> ~/.bashrc
```

---

### **Apply Kinect USB Fix**

```bash
sudo modprobe -r gspca_kinect
```

This command was needed to avoid the `LIBUSB_ERROR_BUSY` issue.

---

## **Kinect Test**

### **Tab 1 - Run Kinect Depth Node**

```bash
cd ~/Ros2-KinectV1
source /opt/ros/humble/setup.bash
source install/setup.bash
sudo modprobe -r gspca_kinect
ros2 run ros2_kinect_depth depth_node
```

---

### **Tab 2 - Check Kinect Topics**

```bash
source /opt/ros/humble/setup.bash
source ~/Ros2-KinectV1/install/setup.bash
ros2 topic list
```

Expected topics:

```text
/kinect/depth/image_raw
/kinect/depth/camera_info
```

---

### **Check Depth Data Rate**

```bash
ros2 topic hz /kinect/depth/image_raw
```

Expected result:

```text
average rate: around 30 Hz
```

---

## **Output / Evidence**

The Kinect v1 sensor successfully published depth data through:

```text
/kinect/depth/image_raw
```

The Kinect depth topic was confirmed to publish at around 30 Hz.

---

## **Problems Faced**

* OpenNI2 did not detect the Kinect v1 properly.
* `ros-humble-openni2-launch` was not available in ROS 2 Humble.
* Kinect USB access sometimes caused `LIBUSB_ERROR_BUSY`.
* The Kinect microphone package caused build issues, but the microphone was not needed for SLAM.

---

## **Solutions**

* Used `KinectV1-Ros2` instead of OpenNI2.
* Used `libfreenect` for Kinect v1 hardware access.
* Skipped the microphone package during build using:

```bash
--packages-skip ros2_kinect_mic_node
```

* Ran the following command before starting the Kinect node:

```bash
sudo modprobe -r gspca_kinect
```
