## **Purpose of This Phase**

The purpose of this phase was to build a 2D map of the environment using the QBot 2 robot.

SLAM Toolbox was used because it worked well with the `/scan` topic generated from Kinect v1 depth data.

The mapping pipeline used:

```text
Kobuki odometry + Kinect depth converted to /scan + TF frames
        ↓
SLAM Toolbox
        ↓
/map
```

---

## **SLAM Toolbox Installation**

```bash
sudo apt install -y ros-humble-slam-toolbox
```

---

## **Full Mapping Pipeline**

The full mapping system was launched using six MobaXterm tabs.

---

## **Tab 1 - Kobuki Driver**

```bash
source /opt/ros/humble/setup.bash
source ~/fyp_ws/install/setup.bash
ros2 launch kobuki_ros kobuki_node-launch.py
```

Purpose:

* Starts the QBot 2 / Kobuki driver.
* Publishes robot odometry on `/odom`.
* Receives velocity commands through `/commands/velocity`.

---

## **Tab 2 - Kinect Depth Node**

```bash
cd ~/Ros2-KinectV1
source /opt/ros/humble/setup.bash
source install/setup.bash
sudo modprobe -r gspca_kinect
ros2 run ros2_kinect_depth depth_node
```

Purpose:

* Starts the Kinect v1 depth node.
* Publishes Kinect depth image data.

Expected topic:

```text
/kinect/depth/image_raw
```

---

## **Tab 3 - Camera Info Fixer**

During actual testing, the camera info fixer script was run from the home directory:

```bash
source /opt/ros/humble/setup.bash
python3 ~/fix_kinect_camera_info.py
```

In the GitHub repository, the script is stored at:

```text
ros2_ws/src/qbot2_perception/scripts/fix_kinect_camera_info.py
```

So it can also be run using:

```bash
source /opt/ros/humble/setup.bash
python3 ros2_ws/src/qbot2_perception/scripts/fix_kinect_camera_info.py
```

Purpose:

* Subscribes to `/kinect/depth/camera_info`.
* Publishes corrected camera info to `/kinect/depth/camera_info_fixed`.

## **Manual Driving for Mapping**

The robot was driven manually using teleop.

```bash
sudo apt install -y ros-humble-teleop-twist-keyboard
ros2 run teleop_twist_keyboard teleop_twist_keyboard --ros-args -r cmd_vel:=/commands/velocity
```

Important note:

The Kobuki base uses:

```text
/commands/velocity
```

not the standard:

```text
/cmd_vel
```

Therefore, the teleop command must be remapped.

## **Problems Faced**

* The robot did not move when teleop was used without remapping.
* SLAM could not work unless `/scan` was publishing.
* Kinect required the `gspca_kinect` module fix before running the depth node.

---

## **Solutions**

* Used teleop remapping:

```bash
ros2 run teleop_twist_keyboard teleop_twist_keyboard --ros-args -r cmd_vel:=/commands/velocity
```

* Verified `/scan` before starting SLAM Toolbox.
* Ran the Kinect USB fix before starting the Kinect depth node:

```bash
sudo modprobe -r gspca_kinect
```
