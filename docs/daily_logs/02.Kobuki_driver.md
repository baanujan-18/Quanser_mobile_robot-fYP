# Day 02 - Kobuki Driver Setup

## **Phase**

**Phase 4 - Install Kobuki Driver for QBot 2 Wheels**

---

## **Work Completed**

* Created the ROS 2 workspace.
* Cloned the required Kobuki/QBot 2 driver packages.
* Installed dependencies using `rosdep`.
* Built the workspace using `colcon`.
* Added the workspace source command to `.bashrc`.
* Connected the QBot 2 to the Raspberry Pi using USB.
* Set USB permission for `/dev/ttyUSB0`.
* Added user `kobuki` to the `dialout` group.
* Launched the Kobuki driver.
* Verified important ROS 2 topics.

---

## **Purpose of This Phase**

The QBot 2 uses a Kobuki mobile base.
This phase connects the robot base with ROS 2 so that the robot can:

* Publish odometry data.
* Receive velocity commands.
* Move using teleoperation.
* Provide wheel feedback for SLAM and navigation.

---

## **Step 1 - Create ROS 2 Workspace**

```bash
mkdir -p ~/fyp_ws/src
cd ~/fyp_ws/src
```

---

## **Step 2 - Clone Kobuki Packages**

```bash
git clone https://github.com/kobuki-base/kobuki_ros.git
git clone https://github.com/kobuki-base/kobuki_ros_interfaces.git
git clone https://github.com/kobuki-base/kobuki_core.git
git clone https://github.com/kobuki-base/velocity_smoother.git
git clone https://github.com/stonier/ecl_tools.git
```

---

## **Step 3 - Install Dependencies**

```bash
cd ~/fyp_ws
sudo rosdep init
rosdep update
rosdep install --from-paths src --ignore-src -r -y
```

Note:

```text
sudo rosdep init
```

is required only once. If it was already done before, it does not need to be repeated.

---

## **Step 4 - Build Workspace**

```bash
cd ~/fyp_ws
source /opt/ros/humble/setup.bash
colcon build --symlink-install
```

---

## **Step 5 - Source Workspace**

```bash
echo 'source ~/fyp_ws/install/setup.bash' >> ~/.bashrc
source ~/.bashrc
```

---

## **Step 6 - Connect QBot 2 through USB**

After connecting the QBot 2 to the Raspberry Pi using USB, the device was checked using:

```bash
ls /dev/ttyUSB*
```

Expected output:

```text
/dev/ttyUSB0
```

---

## **Step 7 - Set USB Permission**

```bash
sudo chmod 666 /dev/ttyUSB0
```

To make serial access easier for the user:

```bash
sudo usermod -aG dialout kobuki
sudo reboot
```

---

## **Step 8 - Launch Kobuki Driver**

After reboot, the Kobuki driver was launched:

```bash
source /opt/ros/humble/setup.bash
source ~/fyp_ws/install/setup.bash
ros2 launch kobuki_ros kobuki_node-launch.py
```

---

## **Step 9 - Verify ROS 2 Topics**

In another terminal:

```bash
source /opt/ros/humble/setup.bash
source ~/fyp_ws/install/setup.bash
ros2 topic list
```

Expected important topics:

```text
/odom
/commands/velocity
/joint_states
```

---

## **Important Working Configuration**

The QBot 2 / Kobuki base uses:

```text
/commands/velocity
```

for robot movement.

It does not directly use the standard:

```text
/cmd_vel
```

Therefore, teleoperation must be remapped like this:

```bash
ros2 run teleop_twist_keyboard teleop_twist_keyboard --ros-args -r cmd_vel:=/commands/velocity
```

---

## **Output / Evidence**

The following topics were verified:

```text
/odom
/commands/velocity
/joint_states
```

This confirmed that the QBot 2 base was connected successfully and the Kobuki driver was working.

---

## **Problems Faced**

* USB permission was required for `/dev/ttyUSB0`.
* Robot movement did not use the normal `/cmd_vel` topic.
* The correct movement topic was `/commands/velocity`.

---

## **Solutions**

* Set temporary USB permission using:

```bash
sudo chmod 666 /dev/ttyUSB0
```

* Added user `kobuki` to the `dialout` group:

```bash
sudo usermod -aG dialout kobuki
```

* Used teleop remapping:

```bash
ros2 run teleop_twist_keyboard teleop_twist_keyboard --ros-args -r cmd_vel:=/commands/velocity
```
