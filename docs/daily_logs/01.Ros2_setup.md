## **Phase 3 - ROS 2 Humble Setup**

### **Set Locale**

```bash
sudo apt install -y locales
sudo locale-gen en_US en_US.UTF-8
sudo update-locale LC_ALL=en_US.UTF-8 LANG=en_US.UTF-8
export LANG=en_US.UTF-8
```

---

### **Add ROS 2 Repository**

```bash
sudo apt install -y software-properties-common
sudo add-apt-repository universe
sudo apt update && sudo apt install -y curl
```

---

### **Add ROS 2 GPG Key**

```bash
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key \
-o /usr/share/keyrings/ros-archive-keyring.gpg
```

---

### **Corrected Repository Command**

The corrected command was used with a space between `dpkg` and `--print-architecture`.

```bash
echo "deb [arch=$(dpkg --print-architecture) \
signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] \
http://packages.ros.org/ros2/ubuntu \
$(. /etc/os-release && echo $UBUNTU_CODENAME) main" \
| sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null
```

Important note:

```text
Correct: dpkg --print-architecture
Wrong:   dpkg--print-architecture
```

---

### **Install ROS 2 Humble**

```bash
sudo apt update
sudo apt install -y ros-humble-ros-base
sudo apt install -y python3-colcon-common-extensions python3-rosdep python3-argcomplete
```

---

### **Configure ROS 2 Environment**

```bash
echo 'source /opt/ros/humble/setup.bash' >> ~/.bashrc
source ~/.bashrc
```

## **Output / Evidence**

The ROS 2 listener successfully received Hello World messages from the talker node.

This confirmed that ROS 2 Humble was installed and working correctly.

---

## **Problems Faced**

* The ROS 2 repository command needed the correct `dpkg --print-architecture` format.
* Without the space, the command would fail.

---

## **Solution**

Used the corrected command:

```bash
dpkg --print-architecture
```

with a space between `dpkg` and `--print-architecture`.
